since Rancher2.x deny to list cluster by name if your token scope is single cluster.
Note: ClusterId is NOT cluster name.

in Rancher2.x "Add API Key" dialog, please check carefully about "Scope" drop-down.
the ClusterId is in brackets after cluster name.
eg: mycluster ( c-h4hxd ), the ClusterId is c-h4hxd

besides, you can find this id in browser address box when you click "Cluster" menu in Rancher2.x Web UI.

if you omit this value, you must specify "clusterName" or "clusterId" in your build arguments.
